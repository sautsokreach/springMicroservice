Circuit Breaker — State Machine (Resilience4j)
STATE
CLOSED
Normal operation.
Calls go through.
Failure rate monitored.
→
failure rate
> 50% (10 calls)
→
STATE
OPEN
All calls BLOCKED.
Fallback method runs.
Waits 5 seconds.
→
after 5s
wait
→
STATE
HALF-OPEN
3 test calls allowed.
Success → CLOSED.
Failure → OPEN again.
SETTINGVALUEMEANING
slidingWindowSize10Monitor last 10 calls to decide state
failureRateThreshold50%Open circuit if ≥ 5 of 10 calls fail
waitDurationInOpenState5sStay OPEN for 5 seconds before trying again
permittedNumberOfCallsInHalfOpenState3Allow 3 test calls in HALF-OPEN state
retry.maxAttempts3Retry failed calls up to 3 times before giving up
retry.waitDuration1sWait 1 second between each retry attempt
timelimiter.timeoutDuration3sTimeout if service doesn't respond within 3 seconds
Zipkin — How a Trace Flows
#SERVICESPANTRACE IDWHAT ZIPKIN RECORDS
1 Gateway :8080 Root span traceId=abc123 Receives request, creates new traceId, forwards to order-service
2 Order Service :8082 Child span traceId=abc123 Receives forwarded request, same traceId propagated via HTTP headers
3 Order Service :8082 Child span traceId=abc123 Calls user-service via @LoadBalanced RestTemplate — span recorded
4 User Service :8081 Child span traceId=abc123 Receives call, same traceId — full hop recorded in Zipkin
5 Order Service :8082 Child span traceId=abc123 Calls product-service via @LoadBalanced RestTemplate — span recorded
6 Product Service :8083 Child span traceId=abc123 Receives call, same traceId — full hop recorded in Zipkin
7 Zipkin UI :9411 Complete trace traceId=abc123 Shows full timeline: Gateway → Order → User + Product with latency per hop
POST /api/v1/orders — Full Request Trace with Circuit Breaker
#FROMTOCALLLAYERWHAT HAPPENS
1 Client Gateway :8080 POST /api/v1/orders Gateway Single entry point. Zipkin creates traceId.
2 Gateway Eureka :8761 Lookup order-service Load Balance lb://order-service → resolved to :8082
3 Gateway Order :8082 Proxied POST Routing TraceId propagated in HTTP headers.
4 Order Service Circuit Breaker Check CB state for user-service Circuit Breaker If OPEN → fallback thrown immediately, skip step 5.
5 Order Service User :8081 GET /api/v1/users/{id} Retry (max 3×) @Retry retries up to 3 times on failure. CB records success/fail.
6 Order Service Circuit Breaker Check CB state for product-service Circuit Breaker If OPEN → fallback thrown immediately, skip step 7.
7 Order Service Product :8083 GET /api/v1/products/{id} Retry (max 3×) @Retry retries up to 3 times on failure. CB records success/fail.
8 Order Service H2 DB INSERT INTO orders Persistence Saved only if both validations passed.
9 All services Zipkin :9411 Push spans Tracing Each service sends its span. Full trace visible at :9411.
10 Gateway Client 200 OK or 503 fallback Response Order JSON returned, or circuit-open error message.

Adds Brave tracer bridge. Every HTTP call automatically gets a traceId + spanId injected into headers and sent to Zipkin.
DEPENDENCY
YML
Zipkin endpoint config (all 4 application.yml)
user-service / order-service / product-service / gateway-service — application.yml

management.tracing.sampling.probability: 1.0 — trace 100% of requests
ZIPKIN
Circuit Breaker + Retry + Fault Tolerance (Order Service only)
POM
spring-cloud-starter-circuitbreaker-resilience4j + spring-boot-starter-aop
order-service/pom.xml
Resilience4j library. AOP is required for @CircuitBreaker and @Retry annotations to work.
DEPENDENCY
CLT
OrderServiceClient.java
order-service/src/main/java/com/microservice/order/client/OrderServiceClient.java
Contains the circuit-breaker logic as a separate Spring bean (required for AOP to intercept calls):
@CircuitBreaker(name="user-service", fallbackMethod="userServiceFallback")
@Retry(name="user-service") on getUser()
Same pattern for getProduct() → product-service
Fallback methods throw a clear error when the circuit is open.
CIRCUIT BREAKER
SVC
OrderService.java
order-service/src/main/java/com/microservice/order/service/OrderService.java
Injects OrderServiceClient and calls serviceClient.getUser() and serviceClient.getProduct() — circuit breaker and retry are transparent.
USES CB
